## Monorepo package layout

The repo uses a **uv workspace** with three Python packages:

| Package name | Directory | Used by |
|---|---|---|
| `sprawl-api` | `api/` | Docker: api container |
| `sprawl-worker` | `worker/` | Docker: worker container |
| `sprawl-shared` | `shared/` | Both api and worker |

The root `pyproject.toml` declares all three as workspace members. Docker images install only what they need (`uv sync --package sprawl-api --no-dev`) to keep images lean.

## The `specs/` rule

`specs/` contains the **design archive** — product definitions, PRDs, tech specs, architecture, and LLD written before implementation. They explain *why* decisions were made.

`docs/` (this site) explains *what exists and how to work with it*. Pages here link to `specs/` for design rationale but do not duplicate the content.
