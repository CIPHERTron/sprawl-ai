---
title: "Environment Variables"
description: "Every environment variable supported by Sprawl AI, with defaults and where each is consumed."
---

All configuration is driven by `.env` at the repo root. Copy `.env.example` to `.env` to get started:

```bash
cp .env.example .env
```

`.env` is loaded by `docker-compose.yml` via `env_file: .env` and read directly by `api/config.py` (Pydantic Settings) for local/out-of-container use.

## PostgreSQL

| Variable | Default | Consumed by |
|---|---|---|
| `POSTGRES_USER` | `sprawl` | `postgres` container |
| `POSTGRES_PASSWORD` | `changeme` | `postgres` container |
| `POSTGRES_DB` | `sprawl` | `postgres` container |
| `DATABASE_URL` | `postgresql+asyncpg://sprawl:changeme@postgres:5432/sprawl` | `api`, `worker`, Alembic |

`DATABASE_URL` must use the `postgresql+asyncpg://` scheme — the codebase uses asyncpg throughout (no psycopg2).

## HashiCorp Vault

| Variable | Default | Consumed by |
|---|---|---|
| `VAULT_ADDR` | `http://vault:8200` | `api`, `worker` |
| `VAULT_DEV_ROOT_TOKEN_ID` | `dev-root-token` | `vault` container (dev mode only) |
| `VAULT_ROLE_ID` | `""` | `api`, `worker` — filled in by `make vault-init` |
| `VAULT_SECRET_ID` | `""` | `api`, `worker` — filled in by `make vault-init` |

<Warning>
  `VAULT_ROLE_ID` and `VAULT_SECRET_ID` are the "secret zero" — the bootstrap credentials. In production these must be injected at runtime (e.g. via a secrets manager or operator), never committed to source control.
</Warning>

## JWT auth

| Variable | Default | Consumed by |
|---|---|---|
| `JWT_SECRET` | `changeme-generate-a-real-secret` | `api` |
| `JWT_ALGORITHM` | `HS256` | `api` |

Generate a production JWT secret with:
```bash
openssl rand -base64 32
```

## LLM — LiteLLM / Ollama (M7+)

| Variable | Default | Consumed by |
|---|---|---|
| `LITELLM_MODEL` | `ollama/llama3.2` | `worker` |
| `OLLAMA_BASE_URL` | `http://ollama:11434` | `worker` |
| `LITELLM_API_KEY` | `""` | `worker` — only needed for hosted LLM providers |

Switch to a hosted model (e.g. GPT-4o) by setting `LITELLM_MODEL=openai/gpt-4o` and `LITELLM_API_KEY=sk-...`.
